## Outils pour créer des maps

| Outil | Type | Avantages | Inconvénients |
|-------|------|-----------|---------------|
| **Papier / tableau blanc** | Physique | Rapide, collaboratif | Pas de versionnage |
| **Wardley Maps Online** | Web | Sémantique Wardley native | Courbe d'apprentissage |
| **Mermaid** | Code (markdown) | Versionnable, intégrable dans la doc | Pas de positionnement 2D natif |
| **draw.io / Excalidraw** | Web / desktop | Flexible, collaboratif | Pas de sémantique Wardley |
| **Figma / Miro** | Web | Atelier remote, templates | Adaptation manuelle |
| **wardleymaps.com (outil intégré)** | Web | Officiel, export | Compte requis |

## Compléments méthodologiques

Les Wardley Maps se combinent bien avec :

| Méthode | Complémentarité |
|---------|----------------|
| **Team Topologies** | Les maps aident à décider quelles équipes gèrent quels composants |
| **Domain-Driven Design (DDD)** | Les bounded contexts DDD peuvent devenir des composants sur la map |
| **Build vs Buy frameworks** | La map fournit le contexte pour les grilles de décision classiques |
| **C4 Model** | La map précède le C4 : stratégie d'abord, architecture ensuite |
| **Lean Startup** | La map aide à identifier le MVP minimum (ne construire que le différenciateur) |

## Pour aller plus loin

1. **Lisez le livre de Simon Wardley** (gratuit en ligne) — chapitres 1 à 5 suffisent pour débuter
2. **Créez 3 maps** de projets différents pour pratiquer
3. **Organisez un atelier** de 2h avec votre équipe sur un projet réel
4. **Révisez vos maps** tous les 6 mois
5. **Partagez** vos maps et décisions — la discussion est la valeur
